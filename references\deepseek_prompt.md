# RAS 坐标转换技能

## 描述
本技能用于将文本形式的 RAS 坐标系数据（包含入口点 `e` 和靶点 `t`）转换为 CSV 格式，并自动为每个坐标分量分配连续的数字后缀（`r1`, `a1`, `s1` 等）。
转换时忽略以 `traj` 开头的行，仅处理 `e` 和 `t` 行。对于俯仰角（P）取负值作为方位角（A）。

## 输入格式示例
```
traj1: R6.89 A5.86 D47.69
e1: R33.90 P25.79 S68.24
t1: R28.21 P30.63 S21.14

traj2: R7.13 P10.33 D49.75
e2: R34.13 P26.53 S67.91
t2: R28.05 P17.68 S19.33

traj3: R2.93 P8.58 D59.33
e3: R34.10 P26.84 S67.62
t3: R31.10 P18.00 S9.02
```

## 转换规则
1. **筛选行**：只保留以 `e` 或 `t` 开头的行（例如 `e1: ...`，`t1: ...`）。
2. **解析数据**：每行格式为 `标签: R数值 P数值 S数值`（数值可为小数）。
3. **坐标映射**：
   - `R` 直接作为右（r）分量
   - `P` 取负值作为前（a）分量（即 `a = -P`）
   - `S` 直接作为上（s）分量
4. **编号规则**：
   - 按照全局处理顺序依次为每个分量分配编号。
   - 每个点（入口或靶点）包含 `r`, `a`, `s` 三个连续编号。
   - 第一个点（e1）使用 `r1, a1, s1`，第二个点（t1）使用 `r2, a2, s2`，依此类推。

## 输出格式（CSV）
每行三列：`点标签, 分量标识, 数值`。
逗号分隔，无表头。

### 输出示例（基于上述输入）
```csv
e1,r1,33.90
e1,a1,-25.79
e1,s1,68.24
t1,r2,28.21
t1,a2,-30.63
t1,s2,21.14
e2,r3,34.13
e2,a3,-26.53
e2,s3,67.91
t2,r4,28.05
t2,a4,-17.68
t2,s4,19.33
e3,r5,34.10
e3,a5,-26.84
e3,s5,67.62
t3,r6,31.10
t3,a6,-18.00
t3,s6,9.02
```

## 注意事项
- 数值保留两位小数（可根据需要调整精度）。
- 确保输入中的 `R`, `P`, `S` 字母大写，并紧接数值（允许空格）。
- 此技能不处理其他格式（如只有 `R A D` 的行）。
